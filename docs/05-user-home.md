# E 用户首页阶段（User Home / Workbench Home）

> 对齐标准：`dsphr_workspace`（鉴权、env）  
> 本文目标：把**工作台首页**的导航结构、入口与 UI 约束写清楚。项目数据与列表见 `07-projects.md`。

**小白说明**：**用户首页**即进入某个项目后的工作台首页：左边是导航（需求书、预算、订单、合同），右边是内容区。设计反馈的入口**不在侧栏**，而在首页/订单页等处的按钮。必须先有**项目**（`project_id`）才能进入工作台，见 `07-projects.md`。

---

## 1. 模块是干嘛的（小白说明）

工作台首页就是打开某个项目后看到的“操作台”首页：左侧导航、右侧内容；首页上会有待办、功能入口卡片等。用户从这里进入需求书、预算、订单、合同；设计反馈则通过页面内按钮进入。

---

## 2. 前端状态结构：Workbench 导航（B 粒度）

来源：如 `src/pages/WorkbenchPage.tsx`

### 2.1 导航 key（现状）

- `NavKey = 'home' | 'requirements' | 'budget' | 'orders' | 'contracts' | 'designFeedback'`

### 2.2 侧栏 `navItems`（现状）

- `home`（项目首页）
- `requirements`（项目需求书）
- `budget`（项目预算）
- `orders`（项目订单）
- `contracts`（项目合同）

**说明**：侧栏**不包含**“设计反馈”；设计反馈通过首页、订单页等处的“进入反馈”等按钮进入。

### 2.3 关键 UI 约束（现状）

- 侧栏 class：`hidden md:flex`  
  **窄屏（手机/小窗口）侧栏会完全隐藏**，用户无法通过侧栏切换 tab，需依赖页面内入口或移动端菜单。
- 默认激活：`active = 'home'`（进入工作台时默认显示首页）。
- 设计反馈入口：项目首页「当前待办：立即处理」、功能入口卡片「设计反馈：进入反馈」、订单页入口等。

#### 小白说明

“项目首页进不去”的常见原因往往是：窄屏下侧栏被隐藏，切到其他页面后找不到入口切回首页。需保证移动端有明确入口回到首页或切换导航。

---

## 3. 与其它阶段的关系

- **上游**：必须有 **07-projects** 的 `project_id`（先选/进入项目，再进入工作台首页）。
- **下游**：首页上的入口分别指向：需求书（与 **06-deep-eval** 需求书展示相关）、预算（**08-budget**）、订单（**08-budget**）、合同（**04-contracts**）、设计反馈（**09-design-feedback**）。
